### 中间件设计
- 服务端中间件类型：`Middleware func(HandlerFunc) HandlerFunc`
  - `HandlerFunc = func(ctx, *nrpc.RpcCaller) error`
  - 内置：AuthMiddleware / LogMiddleware / RecoveryMiddleware
- 客户端中间件类型：`ClientMiddleware func(ClientHandlerFunc) ClientHandlerFunc`
  - `ClientHandlerFunc = func(ctx, message.Header, string, ...[]byte) ([]byte, error)`
  - 内置：ClientLogMiddleware / ClientAuthMiddleware
- 中间件通过 `chainMiddleware` / `chainClientMiddleware` 按顺序包装

### 关键修复
- `channel.go`：`Call()` 签名 `map[string]string` → `message.Header`
- `channel.go`：`TcpChannelServer` 新增 `server *TcpServer` 字段，`handleCall` 通过 `ch.server` 调用 `CallFunc`
- `server.go`：`handler IServerHandleMessage` → `handler wsocket.IServerHandleMessage`
- `server.go`：`NewTcpChannelServer` 传入 `s` 作为参数
- `transport.go`：`Listen` 返回值用 `_, err :=` 接收两个值
- `middleware.go`：`AuthMiddleware` 中用类型断言 `msg.Channel.(nrpc.AuthChannel)` 访问 `GetAuthInfo()`
